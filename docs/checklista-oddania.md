# Checklista oddania zadania lub projektu

Użyj tej listy przed oddaniem pracy albo podczas sprawdzania rozwiązania.

## Kompilacja

- [ ] Program kompiluje się bez błędów.
- [ ] Program kompiluje się z flagami `-Wall -Wextra -pedantic`.
- [ ] Jeśli projekt wymaga kilku plików, instrukcja kompilacji obejmuje wszystkie potrzebne pliki.
- [ ] Dla projektu wieloplikowego działa skrypt build albo opisana komenda.

## Uruchomienie

- [ ] Program uruchamia się z instrukcji w `README.md` albo z opisu zadania.
- [ ] Program działa dla danych poprawnych.
- [ ] Program obsługuje dane graniczne.
- [ ] Program reaguje czytelnie na błędne dane.
- [ ] Program nie kończy się awaryjnie w podstawowych scenariuszach.

## Repozytorium i pliki

- [ ] W repozytorium nie ma plików wynikowych, np. programów `.exe`, plików `.o` albo katalogów build z artefaktami.
- [ ] Pliki źródłowe mają czytelne nazwy.
- [ ] Większy projekt jest podzielony na sensowne katalogi, np. `src`, `include`, `tests`.
- [ ] Dane przykładowe są małe i potrzebne do uruchomienia programu.

## Jakość kodu

- [ ] Nazwy zmiennych, funkcji i klas opisują ich rolę.
- [ ] Funkcja `main` nie zawiera całej logiki większego programu.
- [ ] Powtarzający się kod został przeniesiony do funkcji.
- [ ] Program nie ignoruje błędów wejścia i operacji na plikach.
- [ ] Student potrafi wyjaśnić najważniejsze fragmenty własnego kodu.

## Dokumentacja

- [ ] `README.md` opisuje cel programu.
- [ ] `README.md` zawiera komendę kompilacji.
- [ ] `README.md` zawiera komendę uruchomienia.
- [ ] `README.md` zawiera scenariusze sprawdzenia.
- [ ] Przy projekcie opisano ograniczenia albo pomysły na dalszy rozwój.

## Ćwiczenia pomostowe

- [ ] Rozwiązanie obejmuje pełny wariant podstawowy z opisu ćwiczenia.
- [ ] Student uruchomił scenariusze sprawdzenia z pliku ćwiczenia.
- [ ] Program pokazuje połączenie kilku tematów, np. pętli z walidacją albo klas z polimorfizmem.
- [ ] Przy większym ćwiczeniu kod jest podzielony na funkcje, klasy albo pliki zgodnie z treścią zadania.
- [ ] Student potrafi wskazać, co byłoby następnym rozszerzeniem.

## Projekty zaliczeniowe

- [ ] Wariant minimum działa stabilnie.
- [ ] Rozszerzenia są opisane osobno od wariantu minimum.
- [ ] Rozszerzenia nie psują podstawowych funkcji.
- [ ] Projekt przeszedł [checklistę projektu](../10-projekty/checklista-projektu.md).
- [ ] Projekt można ocenić według rubryki z [10-projekty](../10-projekty/README.md).
- [ ] Przy odbiorze można wypełnić [kartę oceny projektu](../10-projekty/karta-oceny-projektu.md).
- [ ] Student potrafi pokazać przypadek poprawny, graniczny i błędny.
