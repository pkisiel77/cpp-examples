# Szablon projektu

## Cel projektu

Opisz, jaki problem rozwiązuje program i kto mógłby z niego korzystać.

## Wariant minimum

Wypisz funkcje, które muszą działać, aby projekt był zaliczony:

- funkcja 1,
- funkcja 2,
- funkcja 3.

## Rozszerzenia

Wypisz funkcje dodatkowe, które można dodać po wykonaniu wariantu minimum:

- rozszerzenie 1,
- rozszerzenie 2.

## Struktura katalogów

```text
projekt/
├── README.md
├── build.sh
├── src/
│   └── main.cpp
└── tests/
    └── README.md
```

## Kompilacja

```sh
sh build.sh
```

## Uruchomienie

```sh
./build/projekt
```

## Scenariusze sprawdzenia

Przed oddaniem sprawdź:

- podstawowy przypadek użycia,
- błędne dane wejściowe,
- pusty zestaw danych,
- kilka operacji wykonanych po kolei.

## Decyzje projektowe

Opisz krótko:

- jakie dane przechowuje program,
- jakie funkcje albo klasy są najważniejsze,
- czego nie udało się jeszcze zrobić.
